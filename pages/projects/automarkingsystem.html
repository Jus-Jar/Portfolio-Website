Project Overview

The Automated Judge System for Object-Oriented Programming I assignments at UWISTA is designed to evaluate and grade Java programming assignments efficiently and consistently. This system automates the grading process by assessing zipped submissions based on naming conventions, functionality, abstraction, and adherence to object-oriented principles. With a user-friendly graphical interface, lecturers can customize grading rubrics and generate test suites to meet specific assignment requirements. Each submission is evaluated within a strict 5-minute limit, with detailed PDF feedback provided, including test results and flagged issues. By streamlining the grading process, this system ensures fairness, transparency, and time efficiency, benefiting both lecturers and students.

Key Features

  • 1) The system evaluates assignments efficiently and consistently applies evaluation criteria and grading rubrics, reducing grading subjectivity.

  • 2) The system generates meaningful and transparent feedback such as test cases passed, pointing out specific errors and overall grade.

  • 3) The system saves time by assessing/grading assignments within a shorter time frame.

  • 4) The system offers flexibility for lecturers to customize grading criteria, rubrics and assessment parameters

Technology Stack

The Automated Judge System for Object-Oriented Programming I assignments is built using Java as the core programming language for its robustness and suitability for object-oriented design. The system leverages Maven as the build automation and dependency management tool, ensuring streamlined project configuration, dependency resolution, and efficient handling of test cases. Java’s extensive standard libraries and compatibility with Maven allow for the seamless integration of custom test suites and grading criteria. Together, Java and Maven provide a reliable and scalable foundation for developing, testing, and deploying the system to meet the course's grading requirements effectively.
